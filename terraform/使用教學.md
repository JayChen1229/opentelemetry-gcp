# Terraform & Terragrunt 操作手冊

> 本文件專注於 Terraform 基礎架構操作。完整的專案架構與部署教學請參考根目錄的 `README.md`。

---

## 1. 架構

```
terraform/
├── terragrunt.hcl                  # 根設定：GCS backend + provider 自動產生
├── bootstrap/                      # 一次性：建立 tfstate GCS Bucket
├── modules/
│   └── gcp_tenant_project/         # 共用模組：1 module = 1 GCP 專案
└── tenants/
    ├── DEV/
    │   ├── env.hcl                 # DEV 共用變數 (billing account)
    │   ├── otel-collector-dev/     # Collector 專案
    │   └── otel-apps-dev/          # 應用服務專案
    ├── SIT/
    └── PROD/
```

**核心概念**：每個租戶 = 一個子目錄 + 一份 `tenant.yaml`。

---

## 2. 快速開始

### 2.1 Bootstrap（一次性）

```bash
cd terraform/bootstrap
terraform init
terraform apply -var="state_project_id=<你的管理專案ID>"
```

> ⚠️ `terraform.tfstate` 必須 commit 到 Git。

### 2.2 建立新專案

```bash
# 1. 複製範本
cp -r terraform/tenants/DEV/otel-apps-dev terraform/tenants/DEV/<新專案>

# 2. 修改 tenant.yaml
vim terraform/tenants/DEV/<新專案>/tenant.yaml

# 3. 建立
cd terraform/tenants/DEV/<新專案>
terragrunt init
terragrunt plan
terragrunt apply
```

## 3. tenant.yaml 範本

### Collector 專案

```yaml
project:
  id: "otel-collector-dev"
  name: "OTel Collector DEV"
  environment: "DEV"

enabled_apis:
  - "run.googleapis.com"
  - "cloudbuild.googleapis.com"
  - "artifactregistry.googleapis.com"
  - "cloudtrace.googleapis.com"
  - "monitoring.googleapis.com"
  - "logging.googleapis.com"
  - "iam.googleapis.com"
  - "cloudresourcemanager.googleapis.com"

builder_sa:
  name: "service-cicd"
  roles: ["roles/run.admin", "roles/artifactregistry.writer", "roles/logging.logWriter", "roles/storage.admin"]

runtime_sa:
  name: "service-collector"
  roles: ["roles/cloudtrace.agent", "roles/monitoring.metricWriter", "roles/logging.logWriter"]

repo_name: "otel-collector"
labels: { purpose: "observability" }
```

### 應用服務專案

```yaml
project:
  id: "otel-apps-dev"
  name: "OTel Apps DEV"
  environment: "DEV"

enabled_apis:
  - "run.googleapis.com"
  - "cloudbuild.googleapis.com"
  - "artifactregistry.googleapis.com"
  - "cloudtrace.googleapis.com"
  - "monitoring.googleapis.com"
  - "logging.googleapis.com"
  - "secretmanager.googleapis.com"
  - "iam.googleapis.com"
  - "cloudresourcemanager.googleapis.com"

builder_sa:
  name: "service-cicd"
  roles: ["roles/run.admin", "roles/artifactregistry.writer", "roles/logging.logWriter", "roles/storage.admin"]

runtime_sa:
  name: "service-cloudrun"
  roles: ["roles/artifactregistry.reader", "roles/logging.logWriter", "roles/secretmanager.secretAccessor", "roles/cloudtrace.agent", "roles/monitoring.metricWriter"]

repo_name: "otel-demo"
labels: { purpose: "application-services" }
```

## 5. 匯入既有專案

```bash
cd terraform/tenants/DEV/<專案名稱>
terragrunt init

terragrunt import 'google_project.this' '<PROJECT_ID>'
terragrunt import 'google_service_account.cicd_builder' \
  'projects/<PROJECT_ID>/serviceAccounts/<SA>@<PROJECT_ID>.iam.gserviceaccount.com'
terragrunt import 'google_service_account.app_runtime' \
  'projects/<PROJECT_ID>/serviceAccounts/<SA>@<PROJECT_ID>.iam.gserviceaccount.com'
terragrunt import 'google_artifact_registry_repository.docker' \
  'projects/<PROJECT_ID>/locations/<REGION>/repositories/<REPO>'

# 驗證 — 應顯示 0 changes
terragrunt plan
```

## 6. 常用指令

| 指令 | 說明 |
|:-----|:-----|
| `terragrunt init` | 初始化 |
| `terragrunt plan` | 預覽變更 |
| `terragrunt apply` | 執行變更 |
| `terragrunt output` | 查看輸出值 |
| `terragrunt state list` | 列出受管理的資源 |
| `terragrunt destroy` | 刪除（PROD 有保護） |
| `terragrunt run-all plan` | 預覽整個環境（在 DEV/ 下執行） |
| `terragrunt render-json` | 檢查 YAML 解析結果 |
| `terragrunt force-unlock <ID>` | State 被鎖住時解鎖 |

## 8. 安全機制

| 機制 | 說明 |
|:-----|:-----|
| PROD 刪除保護 | `deletion_policy = PREVENT` |
| SA 最小權限 | Builder 只能 `actAs` 指定的 Runtime SA |
| API 安全 | `disable_on_destroy = false` |
| State 版控 | GCS Bucket 開啟 versioning |
| API 等待 | 新專案建立後 `time_sleep` 30s 確保 API 就緒 |
